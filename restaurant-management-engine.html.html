The Governing Axiom

Non-Negotiable Rule

The kitchen receives only orders where payment status is Confirmed. Every other state remains invisible to the kitchen — without exception.

Initial Flow

01 — Session Creation
Table Scan &
Session Birth
  • Customer scans table QR code
  • Unique Session ID & Table ID assigned
  • Status set to Active with timestamp
  • Table status becomes Occupied
  • Order source recorded as QR
02 — Order Building
Cart Construction
& Validation
  • Customer selects items, quantities & add-ons
  • Special instructions captured per item
  • Real-time inventory validation runs
  • Tax & service charges calculated
  • Cart held in session — kitchen uninvolved
03 — Decision Point
Payment
Decision
  • Customer proceeds to payment
  • Final inventory lock runs
  • Order ID created, status → Pending
  • Stock temporarily reserved
  • Three distinct scenarios diverge here
